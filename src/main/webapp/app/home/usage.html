How to use this application

Follow this steps to get a better result

Search the issues you would like to add to the report. Go to Jira -> "Search for issues" and filter the issues you are interested in reporting. Learn more Hide
  • For filtering the issues you can either use the filters provided by Jira or use your own search query.
  • For creating our own search, click on Advanced:


  • Then add your search terms. I will strongly recommend to add the epics to your search, this way you could summarize your report by epics.

    In the example below, we use the query "type=Epic or (sprint=7 and status=Done)" to search for all issues which are epics or all issues included in the sprint 7 and are done.




Export the issues in XML format. Once you have filtered the issues click on "Export" and then "XML" Learn more Hide
  • As you can see in the picture below, on the right corner you have the option to export the list of issues to XML.

Upload the XML file into the application. Learn more Hide
  • Using the XML file from the previous step, upload it in the report report section.

Organize the structure of your word document by adding/removing sections in the "Build the document" section. Learn more Hide
  • Add section clicking on the button or remove a section clicking on
  • The differemt sections that could build in your document are:
    • Epic Summary: Creates a table with a summary of all your epics. Select columns related to each epic and/or summarize by time estimate or spent and/or number of issues related to that epic.

      Note: In order to show epic details you will need to add your epics to the list of issues, you can do that creating an advanced query like in the 1 step.

    • Story Summary: Creates a table with a summary of all your stories. Select columns related to each story and/or summarize by time estimate or spent and/or number of issues related to that epic.

      Note: In order to show epic details you will need to add your stories to the list of issues, you can do that creating an advanced query like in the 1 step.

    • Issues by assignee: Creates a table per person assigned to an issues. Select columns related to an issue.

    • Issues by Epic: Creates a table per epic. Select columns related to an issue.

    • Issues by Story: Creates a table per story. Select columns related to an issue.

    • All the issues: Creates a table with all the issues together.
